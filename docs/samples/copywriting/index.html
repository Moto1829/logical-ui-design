UI Sample

文言は操作と状態を具体化する

抽象語のボタンや曖昧なエラーは、ユーザーに余計な判断コストを発生させます。

学習トップへ サンプル一覧へ
Bad

悪い例

抽象的な文言ばかりで、何が起きるかも何を直すかも分かりません。

設定変更

入力が正しくありません

  • ボタンを押した結果が読めない
  • 何をどう直すべきかが不明である
  • 文脈を読み直さないと判断できない
Good

良い例

操作結果と修正方法が具体的で、次の行動をその場で決められます。

通知先メールアドレスの変更

変更後はこのアドレスに通知が届きます。

メールアドレスの形式で入力してください。

  • 押した結果がボタン文言で分かる
  • 補助説明が不安を減らしている
  • エラー文言が修正方法を返している
