UI Sample

失敗時は回復導線を返す

エラー状態では、問題の通知だけでなく、どう回復できるかを示す必要があります。

学習トップへ サンプル一覧へ
Bad

悪い例

失敗したことだけを伝え、何が起きたかも次の行動も分かりません。

保存に失敗しました

エラーが発生しました。

  • 原因が曖昧で、ユーザーが判断できない
  • 再試行すべきか待つべきか分からない
  • 権限不足なのか通信失敗なのか区別できない
Good

良い例

何が起きたかと、次に取れる行動がセットで示されています。

保存できませんでした

通信が不安定なため保存に失敗しました

入力内容は保持されています。接続を確認して再試行してください。

  • 原因と状態が具体的に伝わる
  • 入力が保持されていることが分かり、不安が減る
  • 次の行動が明確で、行き止まりにならない
