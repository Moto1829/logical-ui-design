UI Sample

アクセシビリティは意味と操作可能性を残す

見た目が整っていても、ラベル、フォーカス、状態説明が欠けると使えないUIになります。

学習トップへ サンプル一覧へ
Bad

悪い例

見た目は軽くても、ラベル不足と色依存で意味が抜け落ちています。

登録フォーム

赤色の項目を確認してください

  • ラベルがなく、入力後に項目の意味が消える
  • エラーが色だけに依存している
  • フォーカス時の強い手がかりが想定されていない
Good

良い例

ラベル、文言、視認しやすい状態表示で意味が途切れません。

登録フォーム

パスワードは8文字以上で入力してください

  • ラベルが常設され、入力中も意味が失われない
  • エラーが文言で説明され、色だけに依存しない
  • フォーカスや状態変化が視認しやすい前提になっている
