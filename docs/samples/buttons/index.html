ボタンの優先順位をどう見せるか

操作の重みが同じように見えると、ユーザーは自分で安全性と優先順位を判断しなければなりません。

学習トップへ サンプル一覧へ
Bad

悪い例

保存、下書き、削除が同じ強さで並び、危険操作も近すぎます。

記事設定

タイトルや公開設定を編集できます。

  • 主要操作と破壊的操作の視覚差がない
  • すべて同じ優先度に見え、判断コストが高い
  • 削除ボタンが近く、誤操作リスクが高い
Good

良い例

主要操作を強調し、補助操作と危険操作を明確に分離しています。

記事設定

変更内容を確認したうえで公開できます。削除は元に戻せません。

  • 主要操作が一目で分かり、行動の軸がぶれない
  • 危険操作が距離と色で分離され、誤操作しにくい
  • 補助操作は使えるが主役になっていない
