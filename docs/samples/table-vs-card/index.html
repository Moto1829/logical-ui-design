UI Sample

テーブルとカードの使い分け

同じ一覧でも、比較したいのか、一件ずつ理解したいのかで適した見せ方は変わります。

学習トップへ サンプル一覧へ
Bad

悪い例

案件比較が必要なのにカードで並べており、項目の横断比較がしにくい状態です。

会員登録導線改善

担当: 田中

状態: 要対応

期限: 4/5

工数: 14h

管理画面改修

担当: 山口

状態: 確認中

期限: 4/7

工数: 9h

  • 状態や期限を横断比較しにくい
  • 件数が増えるほど視線移動が増える
  • 比較目的の一覧として効率が悪い
Good

良い例

比較が必要な情報を列でそろえ、何が違うのかを一目で見られる状態です。

案件 担当 状態 期限 工数
会員登録導線改善 田中 要対応 4/5 14h
管理画面改修 山口 確認中 4/7 9h
  • 列で比較できるため、差分を素早く把握できる
  • 比較したい項目が同じ位置に並び、探索しやすい
  • 一覧上の判断に向いた構造になっている

カードが向く場面

一件ごとの文脈や補足情報が大切で、比較より理解が重要なときはカードが向いています。求人一覧、記事一覧、案件の概要一覧などが典型です。
