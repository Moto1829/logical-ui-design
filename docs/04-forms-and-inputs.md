# 第4章 フォームと入力UI

フォームは、ユーザーに負荷をかけやすいUIの代表です。入力、確認、修正という複数の認知行動が発生するため、設計の良し悪しがそのまま離脱率や誤入力率に直結します。

## 1. ラベルを消さない

placeholderをラベル代わりに使うと、入力開始後に項目の意味が消えます。これは短期記憶への依存を増やすため、誤入力や再確認の原因になります。

原則:

- ラベルは常時見えるようにする
- 必須か任意かを明示する
- 補助説明は入力前に読める位置に置く

特にフォームが長い場合、ラベルが常時見えていることは再確認のしやすさに直結します。ユーザーは一度で完璧に入力するわけではなく、戻って見直しながら入力します。そのときに項目の意味が消えていると、修正の負荷が増えます。

ラベルで意味を保てたら、次に重要なのは入力の流れそのものです。ここからは、ユーザーが無理なく進める並び方を見ます。

## 2. 入力順は思考順に合わせる

ユーザーが頭の中で整理する順番と、フォームの並び順が一致していると入力しやすくなります。

例:

- 個人情報
- 連絡先
- 任意情報
- 同意と送信

入力順序は、システム都合ではなく、ユーザーの頭の中の整理順に合わせるのが基本です。たとえば「確認コード送付のために電話番号が先に必要」という事情があるなら、その理由も近くに示した方が納得しやすくなります。

複数ステップに分ける場合も、途中で何を入力していて、あと何が残っているかが分かるようにします。

## 3. エラーは事後報告ではなく修正支援

「エラーがあります」だけでは足りません。どこが、なぜ、どう直せばよいかが必要です。

良いエラー表示の条件:

- 問題のある項目の近くに表示される
- 原因が具体的である
- 修正方法が分かる
- 色だけに頼らず伝わる

さらに、可能であればエラーの発見は早い方がよいです。送信後にまとめて指摘するより、入力中に明らかな不備が分かるものは早めに伝えた方が、修正コストを減らせます。

悪い例:

- 「入力内容を確認してください」だけが画面上部に出る
- エラー箇所までスクロールしないと分からない

良い例:

- 項目直下に「メールアドレスの形式で入力してください」と表示する
- 画面上部にも要約を出しつつ、各項目にも具体的な説明を出す

## 4. 入力部品は役割に応じて選ぶ

すべてをテキスト入力で済ませると、判断も入力も重くなります。選択肢が決まっているなら、適切な部品を使う方がよいです。

- 少数選択ならラジオボタン
- 複数選択ならチェックボックス
- 候補が多いなら検索可能な選択UI
- 自由記述が必要な場合だけテキスト入力

また、入力フォーマットが決まっている場合は、ユーザーに推測させない工夫も必要です。

- 日付は日付入力UIや明確な形式例を使う
- 電話番号は桁数や区切りを予測しやすくする
- パスワード要件は入力前に示す

これは単に親切というだけでなく、修正回数を減らし、離脱を防ぐ効果があります。

## 5. 送信前の不安を減らす

フォームでは、入力そのものだけでなく、送信してよいかどうかの不安も大きな障壁になります。

不安を減らす方法:

- 送信後に何が起きるかを明示する
- 自動課金や公開範囲など重要な結果を事前に伝える
- 破壊的な結果がある場合は確認を設ける

ユーザーは、分からないから入力を止めることがあります。必要なのは勢いではなく、安心して進める根拠です。

こうした原則があっても、実務では細かな見落としで体験が崩れることがあります。次は、レビュー時に特に確認したい点をまとめます。

## 6. 実務で見落としやすい点

- 必須項目が多すぎて、本当に必要な情報が埋もれていないか
- 任意項目が主要導線を邪魔していないか
- スマートフォンでキーボードが開いたときに操作しにくくないか
- ボタン文言が「送信」「次へ」だけで結果を想像しにくくないか

フォームの悪さは、見た目の印象よりも行動停止として現れます。少しの分かりにくさでも、入力、確認、修正のたびに負荷が積み重なり、途中離脱や誤送信が起きやすくなります。

- 誤入力が増える: ラベル不足や形式不明で、正しい入力を推測する必要がある
- 修正コストが上がる: エラー箇所と修正方法が分からず、やり直し回数が増える
- 送信をためらう: 結果が読めず、公開や課金などの不安で手が止まる
- 離脱につながる: 長い入力の末に失敗や再入力が起きると、完了前に諦めやすい

フォームは小さな迷いが積み重なって離脱に直結しやすい領域です。最後に、この章の要点を振り返ります。

## 7. この章の要点

- フォームでは記憶負荷と修正コストを減らす
- ラベル、補助説明、エラー表示は意味が消えないよう設計する
- 入力順序と入力部品の選択が使いやすさを左右する
- 送信前の不安まで含めて設計する
