# 第1章 UI設計の基礎

UIデザインを見た目だけで評価すると、「なんとなく読みやすい」「なんとなく今っぽい」という曖昧な会話になりがちです。ここでは、UIを評価するための最低限の言葉を揃えます。

## 1. UIは視覚表現ではなく、認知と行動の設計

ユーザーは画面を見た瞬間に、次の流れを短時間で処理しています。

1. どこに何があるかを知覚する
2. その意味を解釈する
3. 何をすべきか判断する
4. 実際に操作する
5. 結果を見て次の行動を決める

UIが悪いと、このどこかで余計な負荷が発生します。つまり、UIデザインの仕事は「情報を飾ること」ではなく、「知覚、理解、判断、操作をスムーズにつなぐこと」です。

## 2. 認知心理学の観点

### 認知負荷を増やさない

人は一度に多くの情報を正確に処理できません。ラベルが曖昧、情報のまとまりが弱い、ボタンの優先順位が不明、といった状態は認知負荷を上げます。

悪い状態の例:

- 項目名がplaceholderにしか書かれていない
- 補足説明とエラー表示が混在している
- 強調すべきボタンが複数あり優先度が分からない

良い状態の例:

- ラベルが常に見える
- 入力項目が意味ごとにグルーピングされている
- 主要行動が視覚的に一番強い

### 視線の流れを設計する

ユーザーは画面全体を均等には見ません。大きさ、位置、余白、コントラストによって視線は誘導されます。重要な情報が散っていると、読解に余計な探索コストが発生します。

### 一貫性で学習コストを下げる

同じ見た目には同じ意味、同じ操作には同じ結果が期待されます。一貫性が崩れると、ユーザーは毎回解釈し直さなければなりません。

## 3. 人間工学の観点

### 押しやすい大きさと間隔

ボタンやチェックボックスが小さいと、特にモバイル環境で誤操作が増えます。タップ対象のサイズと周辺余白は、見た目ではなく身体的な操作しやすさに直結します。

### 読みやすい文字とコントラスト

薄い文字色、小さすぎる文字、詰まりすぎた行間は、それだけで読み取りコストを上げます。UIはポスターではなく、短時間で理解される必要のある道具です。

### フィードバックの明確さ

入力が正しいのか、保存できたのか、何が原因で失敗したのかが分からないUIは不安を生みます。操作結果が即座に明確に返ることは、安心して操作できる条件です。

## 4. 悪いUIと良いUIの見方

悪いUIは、単に見た目が古いUIではありません。次のような状態を指します。

- 情報の優先順位が分からない
- どこを見ればよいか分からない
- 何を入力すればよいか分からない
- 押してよいか不安になる
- 間違えたときに復帰しにくい

良いUIは、その逆です。見た目の装飾量が少なくても、理解しやすく、操作しやすく、結果が予測できるなら良いUIです。

## 5. サンプルで確認する

会員登録フォームを題材に、悪い例と良い例を比較できるサンプルを用意しています。

- [フォームUIの比較サンプル](samples/forms/index.html)

サンプルでは次の違いを確認できます。

- ラベルが常時見えるか
- 入力項目のまとまりが明確か
- 文字の可読性が十分か
- 主要ボタンが明確か
- エラーや補足の扱いが理解しやすいか

## 6. この章の要点

- UIデザインは視覚の好みではなく、認知と行動を助ける設計である
- 認知負荷を減らし、視線誘導と一貫性を整えることが重要である
- 人間工学の観点から、読みやすさと操作しやすさを保証する必要がある
- 良いUIは、速く理解でき、迷わず操作でき、結果が予測しやすい
