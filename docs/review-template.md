# UIレビュー用テンプレート

このテンプレートは、画面レビューを好みではなく根拠で進めるための雛形です。レビューコメントを書くときは、問題、理由、影響、改善案の順で整理します。

## 1. 画面の基本情報

- 画面名:
- URLまたは対象ファイル:
- 主目的:
- 主なユーザー行動:

## 2. 4つの軸で確認する

### すぐ理解できるか

- 画面の目的が一目で分かるか
- 重要情報が最初に見つかるか
- 用語や見出しが一貫しているか

### 迷わず選べるか

- 主操作が明確か
- 選択肢の違いが分かるか
- 情報量が多すぎないか

### 安心して操作できるか

- ボタン文言から結果を予測できるか
- 保存、送信、削除の違いが分かるか
- 処理中や完了後の状態が伝わるか

### 間違えても復帰できるか

- エラー時に回復方法が分かるか
- 破壊的操作に配慮があるか
- 戻る導線や再試行導線があるか

## 3. アクセシビリティ確認

- 色だけに依存していないか
- コントラストは十分か
- フォーカス位置は見えるか
- ラベル、見出し、構造が明確か

## 4. コメント記入テンプレート

### 指摘テンプレート

- 問題:
- 理由:
- ユーザー影響:
- 改善案:

### 記入例

- 問題: 主要ボタンと削除ボタンが近く、同じ強さで表示されている
- 理由: 役割の差が視覚的に伝わらず、誤操作しやすい
- ユーザー影響: 重要な操作の直前に不安が生じ、誤って削除するリスクがある
- 改善案: 削除ボタンを距離と色で分離し、主要ボタンのみ強く見せる

## 5. 優先度の付け方

- 高: 誤操作、離脱、理解不能につながる問題
- 中: 認知負荷が高く、操作時間や迷いを増やす問題
- 低: 品質向上につながるが、直ちに利用不能ではない問題

## 6. 最後に確認すること

- 修正案は画面の主目的を強めているか
- 細部ではなく根本原因に触れているか
- 特定のユーザーだけでなく、全体の使いやすさに寄与しているか
