# 第8章 ダッシュボードの情報優先順位

ダッシュボードは、情報をたくさん置く画面ではありません。ユーザーが「いま何が起きていて、何に先に対応すべきか」を短時間で判断するための画面です。ここでは、ダッシュボードの情報優先順位をどう決めるかを整理します。

## 1. ダッシュボードの役割を誤解しない

ダッシュボードは、持っている情報をすべて並べる場所ではありません。主な役割は次の3つです。

- 状況を一目で把握する
- 異常や変化に気づく
- 次の行動へ素早く移る

この役割を外れる情報を載せすぎると、見た目は賑やかでも、判断の速さは落ちます。ダッシュボードに詳細説明や操作を詰め込みすぎると、一覧画面や詳細画面の役割まで混ざってしまいます。

役割が定まったら、次に決めるべきなのは「誰にとっての最優先か」です。同じ画面でも、利用者が変われば最初に必要な情報は変わります。

## 2. 最初に見せるべきものは何か

ユーザーがダッシュボードを開く理由は、画面によって違います。まず、誰が何を確認するために開く画面なのかを明確にします。

例:

- 運用担当: 異常、未対応、期限切れ
- マネージャー: 進捗、遅延、全体傾向
- 営業担当: 今日の優先案件、対応漏れ、次アクション

つまり、同じデータでも最上段に置くべき情報は利用者によって変わります。最初に表示するカードは、「重要だから」ではなく「最初の判断に必要だから」で決めるべきです。

## 3. KPIとアラートは分けて考える

ダッシュボードでは、定常的に確認したい数字と、今すぐ反応が必要な異常を同じ強さで見せない方がよいです。

分け方の基本:

- KPI: 状況の全体像を把握するための数字
- アラート: すぐ対応が必要な変化や問題

悪い状態:

- 売上、タスク数、エラー件数、未読通知が全部同じ見た目で並ぶ
- 危険な状態が通常情報の中に埋もれる

良い状態:

- 異常や未対応は上部または強い視覚差で分離する
- KPIは比較しやすい配置にまとめる
- 次アクションが必要な項目には導線を持たせる

## 4. カードの並びは視線の順番で決める

カードをどう並べるかは、デザインの好みではなく、視線の流れで決めます。一般的には次の順が扱いやすいです。

1. 異常や注意喚起
2. 今日の判断に必要な主要KPI
3. 詳細確認や掘り下げに使う補助情報

横に並ぶカードの数が増えるほど、視線は散ります。4枚や5枚を同じ強さで並べると比較しづらくなるため、特にモバイルでは情報の塊を小さくする必要があります。

## 5. 情報過多を防ぐ

ダッシュボードは、情報が多いほど価値が高いわけではありません。むしろ、次のような状態は判断を遅らせます。

- グラフ、表、通知、カードが同時に主張している
- 使われないカードが常設されている
- 詳細画面で見るべき内容までダッシュボードに載っている

対策:

- 初見で見なくてもよい情報は折りたたむ
- 補助情報は別画面へ逃がす
- 本当に毎日見る情報だけを残す

ここまで整理すると、良いダッシュボードは何を満たしているべきかが見えてきます。次は、判断の起点として機能する条件をまとめます。

## 6. 良いダッシュボードの条件

良いダッシュボードは、見た瞬間に次の問いへ答えやすい状態です。

- 今、問題は起きているか
- 何を先に見ればよいか
- 今日の優先順位は何か
- 詳細を見るべき項目はどれか

この4つにすぐ答えられないなら、カードの数や優先順位、文言、色の使い方を見直す必要があります。

ダッシュボードの優先順位が崩れると、ユーザーは判断の起点を失います。情報は見えていても、どれに先に反応すべきか分からず、対応遅れや見落としが起きます。

- 緊急対応が遅れる: 重要アラートが通常情報に埋もれ、気づくのが遅くなる
- 日々の判断が鈍る: 何を先に確認すべきか毎回考え直す必要がある
- 詳細確認へ無駄に入る: 一覧で判断できず、複数画面を往復する
- 監視疲れが起きる: 毎回情報の仕分けから始まり、運用負荷が高まる

ダッシュボードは情報量ではなく、最初の判断をどれだけ支えられるかで評価すべきです。最後に、この章の要点をまとめます。

## 7. この章の要点

- ダッシュボードは情報の倉庫ではなく、判断の起点である
- KPIとアラートは同じ強さで扱わない
- カードの並びは視線誘導と次行動で決める
- 情報過多を避け、最初の判断に必要な情報だけを残す
