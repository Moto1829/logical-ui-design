# 第14章 モバイルUIとタッチ操作

モバイルUIでは、画面が小さいことより、操作条件が厳しいことが重要です。片手で持ち、移動中に見て、キーボード表示で画面が狭くなり、通信も不安定かもしれません。ここでは、モバイル特有の制約を前提にどう設計するかを整理します。

## 1. モバイルでは視認性と到達性を同時に考える

デスクトップでは見やすければ十分だった要素も、モバイルでは押しやすさまで考えなければなりません。特に重要なのは次の3点です。

- タップ対象が十分な大きさか
- 親指で届きやすい位置にあるか
- 誤タップしやすい密度になっていないか

小さいリンクや詰まりすぎたボタンは、理解できても操作できません。

## 2. 画面上部と下部では役割を分ける

モバイルでは、画面上部は見出しや状態確認に向きますが、主要操作は下部の方が届きやすいことが多いです。そのため、情報と操作の役割を分けると扱いやすくなります。

例:

- 上部: 画面タイトル、状態、検索結果数
- 下部: 主要CTA、タブ切替、確定操作

逆に、最も頻繁に押すボタンが右上に固定されていると、片手操作では負担が大きくなります。

## 3. モバイルフォームでは入力中の画面変化を考える

フォームでは、ソフトウェアキーボードが表示されると、使える高さが一気に減ります。そのため、入力中に次の項目や送信ボタンが見えなくなる問題が起きやすくなります。

見るべき点:

- ラベルが常に見えるか
- 入力中の文脈が保たれるか
- 次の操作へ進みやすいか

プレースホルダーだけをラベル代わりにすると、入力開始後に何の項目か分からなくなるため、モバイルでは特に不利です。

## 4. スクロール前提でも優先順位は必要である

モバイルでは縦スクロールが前提ですが、だからといって情報の順序を軽視してよいわけではありません。最初に見える範囲に何を置くかは依然として重要です。

基本:

- 最上部には画面の目的と現在地を置く
- 最初の一画面で主要判断ができるようにする
- 補助情報は後半へ送る

「あとで読めばよい」と考えて説明を長く積むと、主要操作が下に追いやられます。

## 5. 移動中や片手操作を前提にする

実際のモバイル利用では、落ち着いた環境で両手操作できるとは限りません。通知や中断も多く、短時間で再開できることが重要です。

そのため、次の点が有効です。

- 状態が途中で失われない
- 入力途中でも再開しやすい
- 1画面内でやることが多すぎない

モバイルの使いやすさは、見た目の省スペース化ではなく、途切れやすい利用に耐えられるかで決まります。

## 6. 良いモバイルUIの条件

良いモバイルUIは、次の問いに答えやすい状態です。

- いま何をする画面か
- 主要操作はどこにあるか
- 片手でも無理なく操作できるか

この3つが明確なら、狭い画面でも迷いと誤操作を減らせます。

モバイルUIが悪いと、理解できるのに操作できない状態が起きます。これは単なる見づらさではなく、片手操作や移動中利用という現実の利用条件に合っていないことが原因です。

- 誤タップが増える: 小さすぎる要素や近すぎる配置で押し間違える
- 入力が中断しやすい: キーボード表示で文脈や送信導線が消える
- 操作を先延ばしにする: 片手ではやりにくく、後でやろうと離脱する
- 再開コストが高い: 途中状態が保たれず、短時間利用に向かない

## 7. この章の要点

- モバイルでは見やすさだけでなく押しやすさも設計対象である
- 画面上部は情報、下部は主要操作という役割分担が有効である
- フォームではキーボード表示後の状態変化を考える
- スクロール前提でも最初の一画面の優先順位は重要である
